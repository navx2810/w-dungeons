Line Parsing

When a new line is added to the campaign, each word (case-insensitive) is checked against the entities.

If the line holds a match, the entry number is stored on the entity.

The matches are highlighted by adding a class to span tag.

Problems

When an entity's name is changed, a background run has to check every entry for the new occurance.

This can be very costly.

Solutions

When a name change occurs, the user can choose to update any lines where the entity was linked to correct the information.

The check would only happen on those lines and should be more efficient.

This would also allow for the user to choose to have the program auto-replace the old name with the new one.

At that point, it's a simple string replace.

Additional Ideas

One idea could be to have each entry clickable. When it is first clicked, the HTML contents are changed to spans with click events attached to them.

When one of the words are clicked, the user can decide whether to link the word to an entity (by creating a new one, or changing the work to represent an already existing entity), or if the word represents an entity, check out the entity.

This means each row has two states.

I'd imagine that only one row can be "highlighted" at a time, so likely the container will handle which one is selected. The formatting can be handled by the entry itself.
